## Priority: Low

### 6. CORS Wildcard Risk

**Current state:** CORS `allow_credentials=True` with explicit origins from `ALLOWED_ORIGINS` env var. Default includes localhost dev ports.
**Risk:** If `ALLOWED_ORIGINS` is misconfigured in production to `*`, credentials would be sent to any origin.
**V1.1 Action:** Add a startup validation that raises on `ALLOWED_ORIGINS=*` when `ENV=production`.

### 7. test_history_shows_logged_count flake

**V1.1 Action:** Investigate and fix the root cause (likely a missing ORDER BY or race in session fixture).

**File:** `tests/unit/test_vdot_continuity.py::TestBreakDetection::test_high_continuity_no_breaks`

### 9. Background Jobs — no retry on failure

**File:** `backend/app/jobs/runner.py`
**Issue:** Failed jobs are logged but not retried. A transient network error causes the job to be marked failed until the next schedule.
**V1.1 Action:** Add exponential backoff retry with max_attempts=3 for transient errors.

### 10. Nutrition search — FCÉN bootstrap required

**File:** `backend/app/integrations/nutrition/`

**V1.1 Action:** Add a startup check that warns if FCÉN table is empty. Document in DEPLOYMENT.md.

The Energy Coach is implemented as pure functions (`energy_patterns.py`, `energy_availability.py`, `allostatic.py`) + graph node rather than a `BaseAgent` subclass. This is a deliberate design decision — energy state detection is stateless computation, not LLM inference. No change needed.

The 8 skipped tests cover behaviours that have been superseded by the LangGraph architecture. They are technically dead test code. Keeping them for documentation value is acceptable for V1.0.
